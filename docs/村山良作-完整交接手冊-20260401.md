## 4. 技術架構

### 4.1 Server（server.py）

Python 標準庫 `http.server`，無額外依賴。

**提供的 API：**

| Method | Endpoint | 用途 |
|---|---|---|
| GET | `/api/articles` | 取得全部作品 |
| POST | `/api/articles` | 新增作品 |
| PUT | `/api/articles/{id}` | 更新作品 |
| DELETE | `/api/articles/{id}` | 刪除作品 |

**靜態檔案：** 同一個 server 也提供所有靜態頁面。

**啟動參數：**
```bash
python3 server.py --port 10814
```

### 4.2 前端

- 純 HTML/CSS/JS，無框架、無建構工具
- 字型：Sora + Noto Sans TC（主站）、Playfair Display + Noto Serif TC + Noto Sans TC（Sort-Hat 頁）
- 風格：Netflix 深色主題
  - `--bg: #141414`（純黑）
  - `--accent: #c4a44a`（金色）
- 作品資料從 `/api/articles` 動態載入

### 4.3 資料管理

作品資料核心是 `data/articles.json`。雖然有 API，但最穩定的維護方式是直接編輯此 JSON 檔。

後台 `admin/index.html` 提供 GUI 管理介面。

## 6. Sort-Hat 分類帽系統

### 6.1 產品說明

婚禮座位查詢互動系統，以哈利波特分類帽為主題。賓客輸入名字 → 看思考動畫 → 得到座位號碼。

- **Demo 網址：** `https://g-skyview.weddingwishlove.com`
- **銷售頁網址：** `https://goodjob.weddingwishlove.com/sort-hat/`
- **後端：** Flask app（獨立服務，非 server.py）
- **資料來源：** Google Sheets（CMS）
- **三個頁面流程：** 選名字 → 思考動畫 → 座位結果（含學院、桌號、座號）

### 6.2 銷售頁設計

銷售頁（`sort-hat/index.html`）是單一 HTML 檔案，內嵌所有 CSS/JS。

**設計規格：**
- Netflix 深色主題（`#141414` 純黑，非紫色系）
- 金色強調色 `#c4a44a`
- 字型：Playfair Display（英文標題）+ Noto Serif TC（中文標題）+ Noto Sans TC（內文）
- IntersectionObserver 捲動動畫
- 手機 mockup 展示（CSS border-radius + ::before notch）

**頁面區塊（由上而下）：**
1. Topbar（品牌 logo + CTA 按鈕）
2. Hero（左文右手機 mockup）
3. Before vs After（情境痛點圖 vs Sort-Hat 解決方案）
4. Timeline flow（3 步驟 + 操作截圖）
5. Features（3 欄功能亮點）
6. Demo phones（交錯手機 mockup 展示）
7. 座位圖重新設計 before/after
8. Pricing（標準版 + 完整版方案）
9. FAQ（5 題，可展開）
10. CTA
11. Footer

### 6.3 方案定價區塊

| 方案 | 包含內容 |
|---|---|
| 標準版 | 分類帽互動系統 + Google Sheets 管理 + 客製化主題 |
| 完整版 | 標準版全部 + 座位圖重新設計 + 大圖輸出一份 |
