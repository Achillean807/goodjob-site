# 村山良作 MVP 落地備忘

更新日期：2026-03-11

## 1. 為什麼先做村山良作

- 目前官網裡最弱，但改完體感最大
- 內容量最少，最容易先收斂
- 不需要先碰複雜商品結構
- 最適合先驗證新版網站語言與 WordPress 落地流程

一句話：
先把 `村山良作` 做成第一個可上線品牌頁，符合 MVP。

## 2. 第一階段只做三頁

1. 首頁
2. 案例總覽頁
3. 合作流程頁

對應原型：

- `prototypes/村山良作-首頁-MVP.html`
- `prototypes/村山良作-案例總覽頁原型.html`
- `prototypes/村山良作-合作流程頁原型.html`

## 3. 官網導流怎麼改

現階段只要做到：

- 官網中「村山良作」入口導到新版首頁
- 新版首頁導到案例總覽頁
- 新版首頁與流程頁都能直接導到聯絡方式

也就是：

`舊官網入口 -> 新首頁 -> 案例 / 流程 -> 洽詢`

## 4. WordPress 最小做法

- 用 3 個靜態頁先上線
- 案例總覽頁先手動挑代表案例，不必先做複雜篩選
- CTA 先接既有聯絡方式
  - Line
  - IG
  - 聯絡表單

## 5. 現有內容怎麼承接

案例總覽頁最適合直接承接你現在 goodjob 專欄裡的文章，不需要重建資料庫。

先分 3 類：

- 品牌活動
- 主題場景
- 展場與空間

每類先挑 2 到 4 個代表案例就夠。

## 6. MVP 不要先做的東西

- 不要先做太多子分類
- 不要先做太多動畫
- 不要先做很重的後台篩選系統
- 不要先補一堆還沒有素材的頁面

## 7. 先上線的判斷標準

只要下面 4 件事成立，就可以先上：

1. 一進頁面就看懂村山良作在做什麼
2. 看得到代表案例入口
3. 看得懂合作流程
4. 找得到洽詢按鈕

## 8. 下一步

如果這組方向成立，下一步不是再畫更多稿，而是：

1. 選定 `村山良作` 正式首頁版本
2. 決定要沿用舊網址還是新 slug
3. 盤點 goodjob 專欄裡要上首頁的代表案例
4. 進 WordPress 落地
