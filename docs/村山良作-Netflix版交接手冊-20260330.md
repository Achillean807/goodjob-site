## 4. 這頁實際怎麼組成
這個頁面不是把卡片硬寫在 HTML 裡。

實際來源是：
1. `blog-netflix/index.html`
   - 只放頁面骨架與 modal 容器
2. `blog-netflix/assets/site.js`
   - 進頁後抓 `/api/articles`
   - 動態 render Top 10 與三個分類列
   - 管 hover popup、hero 切換、modal、lightbox、箭頭
3. `blog-netflix/assets/site.css`
   - 整頁視覺樣式
4. `data/articles.json`
   - 真正的內容資料來源

API 由這支 server 提供：
- `F:\SynologyDrive\Share\39 Ace 秦天霸\村花弄囍官網\03-prototypes\murayama-standalone-site\server.py`

目前 server 會提供：
- `GET /api/articles`
- `POST /api/articles`
- `PUT /api/articles/{id}`
- `DELETE /api/articles/{id}`

但現在實際維護上，最穩定的方式還是直接改 `articles.json`。

## 5. 目前前台互動規格
目前 live 行為如下：

### 5.1 首頁 hero
- 卡片 hover 會切 hero 背景
- 若該篇有 `videoId`，hero 會嘗試播放 YouTube 預覽
- 右下有靜音切換

### 5.2 作品列
- 有 `Top 10 / 商業活動 / 春酒尾牙 / 戶政改造`
- 桌面版每列都有左右箭頭
- 內容本質是水平 rail，不是一次全部攤開

### 5.3 Hover popup
- 滑鼠停在卡片上會出現小視窗
- 有播放鍵、複製連結鍵、展開鍵
- 無影片案例會輪播圖片

### 5.4 大型 detail modal
- 點 popup 的播放鍵會開大視窗
- 上方主區顯示影片或主圖
- 中段顯示分類、標題、說明、外連
- 下方不是舊的「集數」列表
- 下方現在是 `相片藝廊` 水平 row
- 有圖多時可用左右箭頭捲動
- 有些案例圖很少，在桌機寬度下會一次顯示完，所以箭頭 disabled 是正常的

### 5.5 Lightbox
- 點 popup 內縮圖會開 lightbox
- 可左右切圖
- 若內容是影片，會走 iframe 播放

## 6. `articles.json` 怎麼看
每篇文章目前至少會用到這些欄位：

- `id`
- `title`
- `description`
- `category`
- `featured`
- `featuredOrder`
- `heroImage`
- `images`
- `linkUrl`
- `videoId`
- `videoVertical`
- `sortOrder`

欄位用途：
- `category`
  - 只能是 `business`、`party`、`civil`
- `featured`
  - `true` 會進 Top 10 區
- `featuredOrder`
  - 控制 Top 10 排序
- `heroImage`
  - 卡片與 hero 主圖
- `images`
  - detail modal 下方相片藝廊來源
- `linkUrl`
  - 可連到獨立單篇頁，也可先連回舊站專欄
- `videoId`
  - 有值就會在 hero / popup / detail modal 走 YouTube
- `videoVertical`
  - 直式影片要設 `true`
- `sortOrder`
  - 控制分類列排序

## 8. 新增或修改一篇作品的標準做法
### 8.1 只補資料，不做新單篇頁
適合情況：
- 先補數量
- 先讓 Netflix 版有內容
- 單篇頁之後再補

做法：
1. 先準備 `heroImage`
2. 若有多張圖，一起放進 `assets/images`
3. 編輯 `data/articles.json`
4. 補上 `images` 陣列
5. `linkUrl` 先填 WordPress 舊文
6. 部署 `articles.json` 和新圖片

### 8.2 做成獨立單篇頁
適合情況：
- 這篇是主力案例
- 需要站內完整承接

做法：
1. 新建 `pages/*.html`
2. 圖片全部放本地 `assets/images`
3. `articles.json` 的 `linkUrl` 改成站內頁面
4. 部署頁面、圖片、`articles.json`

## 11. 常見坑
### 11.1 看到舊版畫面
大多是快取，不一定是部署失敗。

處理方式：
- 先 `Ctrl+F5`
- 若還不對，檢查 `index.html` 裡引用的 query string
  - 例如：`site.css?v=20260330b`
  - 例如：`site.js?v=20260330b`

### 11.2 終端機看到中文亂碼
這台 PowerShell 常見 `cp950` 顯示問題。

意思是：
- 終端顯示可能亂
- 檔案本身不一定壞
- 前台正常時，不要先誤判成內容毀損

### 11.3 Modal 底部沒看到箭頭
不一定是壞掉。

如果該篇只有少量圖片，在桌面寬度下會一次顯示完：
- row 仍然是水平藝廊
- 只是不用捲
- 所以箭頭會 disabled

### 11.4 圖片有了，但頁面還是舊資料
常見原因：
- `articles.json` 沒上傳
- 新圖檔有上傳，但 `heroImage` / `images` 路徑沒改
- 路徑拼錯
