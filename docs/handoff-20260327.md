## 3. 關鍵檔案說明

### 前端 (blog-netflix/)
| 檔案 | 說明 |
|------|------|
| `index.html` | 主頁面骨架（hero + section shells + popup DOM + lightbox DOM + footer） |
| `assets/site.css` | 全站樣式（含 popup、lightbox、expand、toast、fab、footer） |
| `assets/site.js` | 全站 JS（fetch API → 動態渲染 + popup + lightbox + expand + hero 影片） |
| `assets/weddingwish-logo.png` | 村花弄囍 logo（透明背景、黑字紅囍） |
| `assets/murayama-logo-white.png` | 村山良作 logo（白色，目前未使用） |

### 後端
| 檔案 | 說明 |
|------|------|
| `server.py` | Python stdlib HTTP server，處理靜態檔案 + `/api/*` REST API |
| `data/articles.json` | 所有作品資料 (20 篇)，格式 `{"articles": [...]}` |
| `data/config.json` | 管理員帳密 (sha256 hash) |
| `admin/index.html` | CMS 後台 SPA（純 HTML/JS/CSS inline） |

### API 端點
| Method | Path | Auth | 說明 |
|--------|------|------|------|
| GET | `/api/articles` | ✗ | 取得所有文章（前台用） |
| GET | `/api/images/{id}` | ✗ | 取得作品圖片清單 |
| POST | `/api/articles` | ✓ | 新增作品 |
| PUT | `/api/articles/{id}` | ✓ | 編輯作品 |
| DELETE | `/api/articles/{id}` | ✓ | 刪除作品 |
| POST | `/api/upload/{id}` | ✓ | 上傳圖片 |

## 5. 文章資料結構 (articles.json)

```json
{
  "id": "lativ-magic-platform",       // 唯一 ID，同時對應圖片資料夾
  "title": "Lativ 魔法月台派對",       // 顯示標題
  "description": "簡短描述...",         // popup 顯示文字
  "category": "party",                 // 分類：business / party / civil
  "featured": true,                    // 是否加入精選
  "featuredOrder": 1,                  // 精選排序
  "heroImage": "/assets/images/works/lativ.../hero.jpg",  // 縮圖
  "images": [...],                     // 圖片列表
  "linkUrl": "/pages/xxx.html",        // 作品連結
  "videoId": "YouTube影片ID或null",     // YouTube 影片
  "videoVertical": false,              // 是否直式影片
  "sortOrder": 1                       // 全域排序
}
```

分類對應：
- `business` → 商業活動
- `party` → 春酒尾牙
- `civil` → 戶政改造

## 6. Popup 互動邏輯 (Plan A)

| 按鈕 | 功能 |
|------|------|
| ▶ (Play) | 有影片 → 開燈箱播 YouTube 全螢幕；無影片 → 開燈箱顯示圖片 gallery |
| 🔗 (Link) | 複製作品連結到剪貼簿，顯示 toast「已複製連結」 |
| ˅ (Expand) | 在 popup 內展開/收合完整描述 + 3 欄圖片 gallery |

燈箱操作：
- `Esc` → 關閉
- `←` `→` → 上/下一張圖
- 點擊 × 按鈕 → 關閉

## 7. 已知問題 / 注意事項

1. **Cloudflare 快取** — 改 CSS/JS 後必須更新 query string 版本號，否則用戶會看到舊版
2. **Logo filter** — 用 `filter: invert(1) hue-rotate(180deg)` 讓黑字在深色背景可見，但紅囍顏色不完全精準
3. **圖片路徑** — 作品圖片存在 `/assets/images/works/{article-id}/` 下
4. **server.py 無框架** — 純 Python stdlib，無需 pip install 任何套件
5. **articles.json 格式** — 必須是 `{"articles": [...]}` dict 包 list，不能是裸 list
