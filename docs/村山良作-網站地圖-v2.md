# goodjob Sitemap V2

更新日期：2026-03-11

## 1. Sitemap 目標

這份 sitemap 是根據 `goodjob / 村山良作` 的品牌角色整理：

- 主題場景與品牌活動工作室
- 案例型站點
- 提案式首頁
- 以合作需求與案例類型分流

所以 goodjob 不是單純作品集，而是：

`服務入口 + 案例分類 + 合作流程`

## 2. 使用者的四條主路徑

goodjob 首頁應該優先對應這四種人：

### A. 想找品牌活動合作

進入：

- 品牌活動

### B. 想找主題場景或拍照區

進入：

- 主題場景

### C. 想找展場或公共空間佈置

進入：

- 展場 / 公共空間

### D. 想先看作品再決定

進入：

- 作品案例

## 4. 每個主頁面的角色

### goodjob 首頁

角色：

- 商業提案入口
- 服務分流頁
- 案例入口

首頁最重要的不是鋪很多情緒圖，而是先讓合作方理解：

- 你們做什麼
- 適合哪些專案
- 怎麼合作
- 可以先看哪些案例

### 服務項目

角色：

- 把合作範圍講清楚
- 降低業主判斷成本

### 作品案例

角色：

- 證明執行力
- 依合作型態讓人快速找到 relevant 案例

### 合作流程

角色：

- 把提案到落地的步驟整理清楚
- 讓商業合作更安心

### 關於 goodjob

角色：

- 補品牌理解
- 說清楚為什麼是「村山良作」

### 合作洽詢

角色：

- 收集活動類型
- 收集場地與時程
- 收集預算範圍

## 5. 首頁內部邏輯

goodjob 首頁應該像 proposal deck，不像一般品牌首頁。

建議首頁順序：

1. Hero：主題場景與品牌活動工作室
2. 服務分流
3. 案例類型入口
4. 合作流程
5. 精選作品
6. 最終洽詢 CTA

## 6. 與現有網站相比，最重要的改變

目前網站的問題不是沒有作品，而是：

- 服務沒有被整理成可判斷的分類
- 案例缺乏入口層次
- 合作流程沒有被清楚說明

Sitemap V2 的目的就是把這三件事補起來。
