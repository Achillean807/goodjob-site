## 9. 維護時的標準做法
### 9.1 改 SEO/AEO 前先判斷頁面層級
- 首頁 / 服務頁：優先做完整 metadata + schema
- 案例頁：先補 canonical + OG，再補 schema
- 純互動頁：至少要能被分享、被索引、被 sitemap 收錄

### 9.2 新增頁面時要同步做
新增一頁時，至少同步補：
- `<title>`
- `<meta name="description">`
- canonical
- OG
- Twitter Card
- apple-touch-icon
- 若屬於重要頁，補 JSON-LD
- 若要被搜尋引擎發現，更新 sitemap

### 9.3 內容更新時要同步做
- 若 URL 不變，只要更新 sitemap 的 `<lastmod>`
- 若更換主圖，也要同步改 OG image

## 11. 驗證方式
### 11.1 原始碼檢查
至少檢查：
- `canonical`
- `og:title`
- `twitter:card`
- `application/ld+json`

### 11.2 靜態檔檢查
- `robots.txt` 可開
- `sitemap.xml` 可開
- `apple-touch-icon.png` 回 `200`

### 11.3 前台檢查
這些不一定有明顯視覺變化，但仍要檢查：
- 分享到 LINE 是否有預覽
- FAQ 頁是否仍正常顯示
- `v2` 與正式站沒有混用錯誤資產路徑
