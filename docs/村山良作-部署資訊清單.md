# 村山良作-部署資訊清單

更新日期：2026-03-22

## 四個關鍵點怎麼串在一起

### 1. `100.102.51.64`
這是遠端主機 `ACH-ClawHome` 的 Tailscale IP。
可用 `ssh achilean@100.102.51.64` 連進 Ubuntu 主機。

### 2. `goodjob.weddingwishlove.com`
這是對外公開網址。
它不是直接指到 WordPress，而是透過 Cloudflare Tunnel 導到遠端主機上的獨立靜態站。

目前已確認的 ingress：
- `goodjob.weddingwishlove.com -> http://localhost:10814`

### 3. `/srv/weddingwish/murayama-goodjob-site`
這是遠端主機上的站點根目錄。
目前公開的 `goodjob` 靜態檔都放在這裡。

已確認內容包含：
- `index.html`
- `cases.html`
- `archive.html`
- `workflow.html`
- `pages/`
- `assets/`

### 4. `/v2/ 404`
`https://goodjob.weddingwishlove.com/v2/` 目前回 `404`。
原因不是服務壞掉，而是：
- 站點根目錄下沒有 `v2/` 這個資料夾
- Cloudflare Tunnel 正常
- `murayama-goodjob.service` 正常
- `goodjob.weddingwishlove.com/` 本身回 `200`

也就是說：
- `goodjob.weddingwishlove.com/v2/` 是不存在的靜態路徑
- 它和 WordPress REST API 的 `wp-json/wp/v2/` 無關

## 實際部署鏈路
```text
Browser
-> goodjob.weddingwishlove.com
-> Cloudflare Tunnel (cloudflared)
-> http://localhost:10814
-> murayama-goodjob.service
-> python3 -m http.server
-> /srv/weddingwish/murayama-goodjob-site
```

## 已確認狀態
- 遠端主機：`ACH-ClawHome`
- SSH：可連
- `murayama-goodjob.service`：`active`
- `cloudflared.service`：`active`
- `goodjob.weddingwishlove.com/`：`200`
- `goodjob.weddingwishlove.com/v2/`：`404`
- 遠端站點目錄存在：是
- 遠端 `v2/` 目錄存在：否

## 目前還缺的 3 個欄位
1. 主站入口何時切換到 `https://goodjob.weddingwishlove.com/`
2. 誰負責後續持續移植剩餘 `goodjob` 文章
3. `/v2/` 這個路徑是不是原本有人預期要存在
   - 如果是，就要補資料夾或 rewrite
   - 如果不是，就應該把所有對 `/v2/` 的口語描述和誤會註記清掉

## 補充：不要混淆的兩個 v2
- `https://goodjob.weddingwishlove.com/v2/`
  - 這是獨立站公開路徑的一部分
  - 目前不存在，所以 404
- `https://www.weddingwishlove.com/wp-json/wp/v2/`
  - 這是 WordPress REST API base
  - 目前可用
